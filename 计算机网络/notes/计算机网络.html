# 一、计算机网络概念

计算机网络：是一个将分散的、具有独立功能的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统。

# 电路交换

# 电路交换的缺点：

如果用户在编辑内容，则占用了通信资源，但通信资源未被利用，所以采用分组交换。

# 分组交换

# 最重要的分组交换机：路由器

发送方：构造分组、发送分组 （添加首部）

路由器：缓存分组、转发分组

接收方：接收分组、还原报文

随时发送分组，无需建立连接，构成原始报文的一个个分组，依次往各 结点交换机 上存储转发，各 结点交换机 在发送分组的同时，还缓存接收到的分组

# 报文交换

① 报文交换中的交换结点也采用存储转发方式，但报文交换对报文的大小没有限制，这就要求交换结点需要有较大的缓存空间

② 报文交换主要用于早期的电报通信网，现在通常被较先进的分组交换方式所取代

# 三种交换方式的优缺点

电路交换优点原因：
    1.建立连接后数据直达
    2.通信双方之间只有一条专用的通信线路
    3.不同的通信双方拥有不同的信道，不会出现争用物理信道的原因
    4.既适合传输模拟信号，也适合传输数字信号
    5.得益于通信时延小的优点
    6.控制简单
缺点原因：
	4.不同类型、不同规格、不同速率的终端很难相互进行通信，难以在通信过程中进行差错控制
	
报文交换优点的原因
    2. 发送方将报文传送给结点交换机时，结点交换机先存储整个报文，然后选择一条合适的空闲线路，将报文发送出去
    3.如果某条路径存在故障，会重新选择另一条路径传输数据
    4.通信双方不是固定占用一条通信线路，而是在不同的时间分段部分占用物理线路
    5.一个报文可以同时发送多个目的地址
缺点原因：
	3.报文需要携带目标地址、源地址等信息
	
分组交换优点的原因
	1.不需要为通信双方预先建立一条专用的通信线路
	2.通信双方不是固定占用一条通信线路，而是在不同的时间分段部分占用物理线路
	3.分组的长度固定，相应的缓冲区的大小也固定
	4.分组是逐个传输的，所以后一个分组的存储操作和与前一个分组的转发操作可以同时进行
	5.分组比报文小，出错概率降低，即使出错也只需重传出错的分组
缺点原因
	3.对于数据报服务时，可能出现失序、丢失或重复分组，分组到达目的结点时，需要重新还原成原始报文，比较麻烦；若分组交换采用虚电路服务，虽然没有分组失序问题，但有呼叫建立、数据传输和虚电路释放三个过程  （虚电路服务大厂问到了）

# MAC地址、IP地址以及ARP协议

MAC地址：在每个主机发送的帧中必须携带标识发送主机和接收主机的地址
		①MAC地址一般被固化在网卡（网络适配器）的电可擦可编程只读存储器EEPROM中，因此MAC地址也被称为硬件地址
		②MAC地址有时被称为物理地址。但这并不意味着MAC地址属于网络体系结构中的物理层
IP地址：①网络编号：标识因特网上数以百万计的网络
	   ②主机编号：标识同一网络上不同主机（或路由器各接口）
	   192.168.0.254
	   192.168.0.1
	   其中192.168.0是网络编号，后面的十进制是主机编号
	   数据包转发过程中源IP地址和目的IP地址保持不变，而源MAC地址和目的MAC地址逐个网络改变
ARP协议：从IP地址得知MAC地址

# IP数据报的发送和转发过程

IP数据报的发送和转发过程包含以下两个部分
一、主机发送IP数据报
二、路由器转发IP数据报 
	IP数据报传给默认网关，默认网关帮主机将IP数据报转发出去

源主机如何知道目的主机是否与自己在同一个网络中？如果在同一个网络那么不需要路由器，可直接交付，否则需要路由器间接交付
    源主机的子网掩码和目的主机的IP地址相与得到的目的网络地址 ！= 源主机的网络地址
    那么源主机和目的主机就不再同一个网络中
主机如何知道路由器的存在？
	路由器将自身 IP地址 给其接口直连网络的各个主机作为 默认网关

# IPv4数据报的首部格式

总长度 = 首部 + 数据
标识，在同一数据的分片时相同
标志：占三位，第一位叫MF，MF=1表示后面还有分片，MF=0表示这是数据报片中最后一个
	第二位是DF，(Don’t Fragment)，意思是不能分片，所以DF=0时才允许分片
片偏移：较长的分组在分片后，某片在原分组中的相对位置
生存时间 ：防止IP数据报在网络中永久兜圈 
协议：占8比特，用于表示IPv4数据报的数据部分是何种协议数据单元
首部校验和：占16比特，用来检测首部在传输过程中是否出现差错，又称因特网校验和

# TCP流量控制

ACK：TCP报文段首部中的标志位，取值1表示这是一个TCP确认报文段
ack: TCP报文段首部中的确认号字段，取值201表示序号201之前的数据已全部正确接收，现在希望收到序号201及其后续数据
rwnd：TCP报文段首部中的窗口字段，取值300表示自己接收的窗口大小为300
流量控制：让发送方的发送速率不要太快，要让接收方来得及接收
利用 滑动窗口 机制可以很方便地在TCP连接上实现对发送方的流量控制
	一、TCP接收方利用自己的接收窗口的大小来限制发送方发送窗口的大小
	二、TCP发送方收到接收方的零窗口通知后，应启动持续计数器。持续计时器超时后，向接收方发送零窗口探测报文
